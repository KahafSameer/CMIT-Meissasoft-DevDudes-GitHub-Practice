Git & GitHub Commands Reference

Basics

git init

Initialize a new Git repository in the current directory.

git add .

Stage all changes in the working directory for the next commit.

git commit -m "message"

Commit staged changes with a descriptive message.

git status

Show the status of changes as untracked, modified, or staged.

git log

Display the commit history of the repository.

git diff

Show differences between working directory and staged changes.

Branching

git branch

List all branches in the repository.

git branch branch-name

Create a new branch with the specified name.

git checkout branch-name

Switch to the specified branch.

git checkout -b branch-name

Create and switch to a new branch.

git merge branch-name

Merge the specified branch into the current branch.

git branch -d branch-name

Delete the specified branch.

Stashing

git stash

Save uncommitted changes for later use.

git stash list

List all stashed changes.

git stash apply

Apply the most recent stash to the working directory.

git stash drop

Remove the most recent stash.

git stash pop

Apply and remove the most recent stash.

Collaboration

git clone url

Clone a repository from a remote URL.

git remote add origin url

Add a remote repository as origin.

git push origin branch

Push commits to the remote repository.

git pull origin branch

Fetch and merge changes from the remote repository.

git fetch origin

Fetch changes from the remote without merging.

git remote -v

List all remote repositories.

Recovery

git reset --soft HEAD~1

Undo the last commit but keep changes staged.

git reset --hard HEAD~1

Undo the last commit and discard changes.

git revert commit-hash

Create a new commit that undoes changes from a specific commit.

git reflog

Show a log of all recent actions in the repository.

git checkout -- file

Discard changes to a specific file.

GitHub Actions

name: CI

Define the name of a GitHub Actions workflow.

on: [push]

Trigger the workflow on push events.

jobs:

Define jobs in a workflow.

uses: actions/checkout@v2

Checkout repository code in a job step.

run: npm test

Run a command in a job step.

CI/CD

deploy:

Define a deployment job in CI/CD pipeline.

environment: production

Specify the environment for deployment.

needs: build

Specify job dependencies in the pipeline.

if: github.ref == 'refs/heads/main'

Conditional execution based on branch.

secrets: ${{ secrets.API_KEY }}

Use secrets in CI/CD workflows.

Interview

What is Git and why is it important for DevOps?

Git is a distributed version control system that tracks changes in source code. It's crucial for DevOps as it enables collaboration, version tracking, and integration with CI/CD pipelines for automated deployments.

How do you create and switch to a new branch in Git?

Use 'git checkout -b new-branch' to create and switch to a new branch, or 'git branch new-branch' followed by 'git checkout new-branch'.

Explain the difference between git merge and git rebase.

Git merge combines branches by creating a merge commit, preserving history. Git rebase rewrites history by applying commits on top of another branch, resulting in a linear history.

What are GitHub Pull Requests and how do they facilitate collaboration?

Pull Requests (PRs) allow developers to propose changes, review code, and discuss modifications before merging. They enable code reviews, automated testing, and collaboration in teams.

How do you handle merge conflicts in Git?

Resolve conflicts by editing the conflicted files, marking resolved sections with ' <<<<<<<', '=======' , and '>>>>>>>' markers, then staging and committing the changes.

What is GitLab CI/CD and how does it differ from GitHub Actions?

GitLab CI/CD is an integrated tool for continuous integration and deployment within GitLab. It differs from GitHub Actions by being more tightly integrated with GitLab's ecosystem, offering built-in runners and a unified UI.

How do you revert a commit in Git?

Use 'git revert commit-hash' to create a new commit that undoes the changes of the specified commit, preserving history. For a hard reset, use 'git reset --hard commit-hash' (destructive). 

What are Git hooks and how are they used in DevOps?

Git hooks are scripts that run at certain points in the Git workflow (e.g., pre-commit, post-merge). In DevOps, they're used for code quality checks, linting, and triggering CI/CD pipelines.

How do you set up a remote repository in Git?

Use 'git remote add origin https://github.com/user/repo.git' to add a remote, then 'git push -u origin main' to push and set upstream.

Explain the concept of Git stashing and when to use it.

Git stash temporarily saves uncommitted changes. Use it when you need to switch branches or pull changes without committing incomplete work, then apply with 'git stash pop'.
